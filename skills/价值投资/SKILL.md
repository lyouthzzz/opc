# 价值投资

## 概述

本 skill 提供一套基于段永平/巴菲特价值投资理念的企业价值分析框架，核心是"right business, right people, right price"三要素判断，辅以五维度深度分析（商业模式、市场规模、竞争壁垒、企业文化、估值）。

两部分设计：
- **通用分析框架**（零公司耦合）：七步分析流程 + 方法论参考，适用于任何公司
- **公司知识库**（用户私人沉淀）：`references/companies/` 下存放用户自己对各公司的深度理解，分析时按需加载作为参照。这是用户主动维护的知识积累，**不是每次分析的产物**——日常分析直接用通用框架完成即可，无需写入知识库。

框架之外，本 skill 优先通过 **通达信 MCP（`tdx`）**获取市场现状数据；`tdx` 没有、失败或字段不足时，再用 **Tushare 兼容 MCP（`tushareMcp`）**补充股价、K 线、财报三大表、财务指标、估值、分红、股东、公告/研报、资金流、宏观等数据。MCP 工具参数与查询模式见 `references/mcp-tool-guide.md`。

## 触发条件

当需要执行以下任务时使用本 skill：
- 分析某家上市公司的投资价值
- 评估企业的商业模式和护城河
- 撰写投资分析报告或投资备忘录
- 判断企业的长期可持续性
- 进行估值分析
- 查询/补充公司的市场现状（最近股价、K 线、年报/财报三大表、财务指标、估值、分红、资金流、宏观环境）
- 用户提到"价值分析""投资分析""right business""段永平框架""市场现状""最新股价""财报""财务数据"等关键词

## 分析流程

执行价值分析时，按以下七步顺序推进。每一步的详细方法论参见 `references/value-investing-framework.md`；市场现状分析所用的 MCP 工具参见 `references/mcp-tool-guide.md`。

分析前，可先检查公司知识库（见下文"公司知识库"章节）是否已有该公司的理解。若有则加载作为参照；若无，直接用通用框架独立分析即可——**分析本身不依赖知识库，也不产生知识库内容**。

### 第零步：锚定数据时效（必做，先于一切 MCP 拉数）

**禁止**用对话记忆、知识库摘录或硬编码年份（如 `period="20241231"`）代替实时查询。每次执行分析必须先确立三个时间锚点，并在后续每一步校验返回数据是否匹配：

| 锚点 | 含义 | 如何获取 |
|------|------|----------|
| **分析基准日** | 本次分析执行的日历日 | 当前日期（如 2026-07-03） |
| **最近交易日** | 行情/估值数据的截止日 | `trade_cal`（`exchange="SSE"`，取 `is_open=1` 且不晚于分析基准日的最大 `cal_date`） |
| **最新报告期** | 财报/财务指标的 `period` | `disclosure_date` → 若无则 `fina_indicator`/`income` 取最大 `end_date`；有 `express`/`forecast` 则优先纳入 |

**时效校验规则**（每条 MCP 返回后执行）：

1. **行情类**（`daily`/`daily_basic`/`hk_daily`/`us_daily`）：`trade_date` 必须等于最近交易日；若落后 >3 个交易日，扩大 `end_date` 重查或标注「行情滞后」
2. **财报类**（三大表/`fina_indicator`/`hk_*`/`us_*`）：`period` 必须等于已确定的最新报告期；若仅有更早报告期，查 `forecast`/`express`/`anns_d` 补业绩快报/预告
3. **事件类**（`anns_d`/`research_report`/`report_rc`）：`ann_date`/`report_date` 取近 90 天内最新；研报/一致预期注明发布日期
4. **宏观类**（`yc_cb`/`index_daily`）：`trade_date`/`date` 不晚于最近交易日超过 5 日

**产出**：在「市场现状快照」开头固定输出 **数据时效声明**：

```
数据时效：分析基准日 YYYY-MM-DD | 行情截止 YYYYMMDD | 财报报告期 YYYYMMDD | MCP 拉取时间 YYYY-MM-DD HH:MM
```

知识库文件仅作定性参照，**不得**替代上述 MCP 实时数据；加载知识库时须标注文件内 `数据截止` 字段，若早于最新报告期则提示「背景理解可能滞后，以 MCP 为准」。

详细工具调用与边界处理见 `references/mcp-tool-guide.md` 第二节「数据时效性校验」。

### 第一步：看懂生意本质

分析任何公司的前提是真正"看懂"这家公司。执行以下子步骤：

1. **破除误解**：明确公司"不是什么"。大多数市场争议源于对公司的错误定义。
2. **定义本质**：用一句话概括公司"是什么"。关注核心需求而非表面产品形态。
3. **识别需求**：公司满足的是什么样的真实需求？是功能性需求还是情绪/精神需求？消费本质上只解决两个问题——满足感和存在感。
4. **判断可理解性**：是否真正看懂了这个生意？如果看不懂，可以 skip——"人生很忙，看不懂的东西是可以 skip 的"。不必勉强投资看不懂的公司。

### 第二步：市场现状分析（金融数据 MCP 支撑）

在定性判断之前，先用 **通达信 MCP（`tdx`）**拉取该公司的市场现状数据，建立客观事实基础。若 `tdx` 没有对应数据、K 线/F10 返回异常、字段不足，或需要港美股/宏观等 `tdx` 不覆盖的数据，再用 **Tushare 兼容 MCP（`tushareMcp`）**补充，不要停在工具错误上。参数、代码格式与查询模式详见 `references/mcp-tool-guide.md`。

**前置条件**：已完成第零步，手头有「最近交易日」与「最新报告期」两个锚点；所有 `trade_date`/`end_date`/`period` 参数均由此推导，**禁止写死年份**。

**采集内容（A 股个股标准顺序，日期一律 `YYYYMMDD`，报告期用 `period`）：**

1. **大盘环境** —— `index_daily(ts_code="000001.SH", end_date=最近交易日)` + `index_dailybasic(trade_date=最近交易日)`：判断市场冷热与指数估值
2. **最近股价 / 历史位置** —— `daily(ts_code, end_date=最近交易日)` 看最新价；`monthly(ts_code, 5年至最近交易日)` 看历史位置
3. **当前估值** —— `daily_basic(ts_code, trade_date=最近交易日)`：PE、PB、换手率、总市值、股息率
4. **财务指标** —— `fina_indicator(ts_code, start/end=近3年至最新报告期)`：ROE、毛利率、净利率、成长/偿债/营运能力
5. **年报三大表** —— `income` + `cashflow` + `balancesheet`（`ts_code, period=最新报告期`）
6. **分红 / 主营 / 指引 / 股东** —— `dividend`、`fina_mainbz`、`forecast`/`express`、`top10_holders`/`stk_holdernumber`
7. **公告 / 研报 / 一致预期** —— `anns_d`（含 PDF）、`research_report`、`report_rc`（盈利预测/目标价）
8. **（DCF 需要时）无风险利率** —— `yc_cb(ts_code="1001.CB")` 国债收益率曲线

**代码格式**：A 股 `600519.SH`/`000001.SZ`，港股 `09992.HK`，美股见 `us_basic`。本 MCP 无"名称→代码"模糊搜索，先用 `stock_basic`/`hk_basic`/`us_basic` 定 `ts_code`。

**市场支持边界**：A 股财务最完整（`income`/`balancesheet`/`cashflow`/`fina_indicator`/`daily_basic` 等）；**港股用 `hk_income`/`hk_balancesheet`/`hk_cashflow`/`hk_fina_indicator`，美股用 `us_income`/`us_balancesheet`/`us_cashflow`/`us_fina_indicator`**——港美股无 A 股式 `daily_basic`，估值指标需据财报与市值自算。**部分接口可能受套餐/权限限制**，权限不足时降级跳过并标注。

**产出**：一份「市场现状快照」（含第零步**数据时效声明** + 当前价、历史位置、估值、最新财报核心指标、分红、公告/一致预期、大盘/宏观背景；每项附 `trade_date`/`period`/`ann_date`），作为第三步 Right Business（财务指标）与第五步 Right Price（估值/安全边际）的数据输入。

> MCP 数据用于**验证**定性判断，不替代"看懂生意 + 五维度 + 四种好运"框架。数据仅供参考，不构成投资建议。

### 第三步：Right Business —— 商业模式分析

围绕三个子维度展开：

**商业模式**：
- 公司靠什么赚钱？收入结构如何（产品/服务/授权占比）？
- 毛利率、净利率水平及变化趋势
- 利润"含金量"：经营性现金流是否远超净利润？账上现金是否充足？有息负债率是否极低？
- 是否轻资产运营？资本回报率（ROE/ROIC）如何？
- 是否具备定价权？产品价格不敏感度如何？

**市场规模**：
- 行业总规模及增速
- 公司市占率及变化趋势
- 天花板在哪里？还有多大空间？
- 国内 vs 海外市场结构及增速差异
- 市场增长驱动是否从"扩张驱动"转向"精细化运营驱动"

**竞争壁垒**：
- 护城河来源（品牌、网络效应、规模、转换成本、独占性资源等）
- 壁垒的宽度和持久性——"真正的强者具有独占性"
- 竞争者能否复制？需要多少资源？达到同样效果需要多长时间？
- 是否存在"黑洞效应"（行业最优质的资源持续向公司汇聚，形成正反馈）？
- 壁垒不能保证需求永远存在，但可以让有需求的人持续关注公司

### 第四步：Right People —— 企业文化与管理层

**管理层评估**：
- 创始人画像：性格沉稳？喜怒不形于色？具备行业创业者的优良品格？
- 是否始终"做对的事情"（do the right things），而非只关注短期利益？
- 是否具备长期主义思维？"相信时间的力量，慢慢来、慢慢做，但要做就要做好、做对"
- 是否敢于否定自己、不被沉没成本束缚？
- 在高光时刻是否保持清醒？是否主动"降温"、控制节奏？
- "华丽的数字背后，其实都意味着陷阱"——是否能看到运营中的问题？

**企业文化**：
- 企业的核心价值观是什么？是利润导向、客户导向还是消费者导向？（消费者导向最纯粹）
- 文化是否能筛选出合适的人？"招人的时候就要招合适的人，合适的人关注对错"
- 管理层在高增长期是否选择"慢下来"夯实内功？是否提出"维修站换胎"式的休整？
- never out of control——放权但不失控，核心靠企业文化

**信号观察**：
- 是否给业绩指引？（段永平认为：不给指引更好。指引对业绩成长无益，且一旦达不到会导致股价大跌和官司。巴菲特的 BRK、网易、拼多多都不给指引）
- 面对质疑如何回应？是情绪化反驳还是用事实和经营说话？
- 资本配置能力：回购、分红、投资节奏是否理性？"本分经营"
- 扩张节奏是否克制？"小步快跑、不行就砍"的产品思维

### 第五步：Right Price —— 估值分析

**估值方法**（由简到精）：

1. **一眼胖瘦法**：
   - "用合理的价格买入一家伟大的公司，远胜于用便宜的价格买入一家平庸的公司"
   - 先判断"胖瘦"（明显高估/低估/合理），再精算
   - 市场给低 PE 不一定是因为"贵"，可能是因为"太悲观"——需辨别

2. **"撇去爆款"模型（极端保守假设）**：
   - 假设剔除公司最核心的爆款/增长引擎的全部利润
   - 假设企业不再成长（零增长）
   - 在此最悲观假设下计算 PE
   - 若极端保守 PE 与合理 PE（如 25 倍）仅一线之隔，说明市场过度悲观，存在安全边际
   - 现实中爆款不会突然归零，其他业务仍在增长，安全边际更加明显
   - 适用于有明确核心增长引擎的公司；多引擎驱动的公司可对每个引擎分别做敏感性分析

3. **多情景估值**：
   - 悲观情景：核心驱动因素回落至爆发前水平 + 零增长
   - 中性情景：管理层指引增速延续
   - 乐观情景：多引擎驱动，新业务/新市场接力
   - 对比当前市值在各情景下的 PE，判断市场定价隐含了哪种预期

4. **安全边际判断**：
   - 当前 PE 隐含了市场什么预期？过度悲观还是过度乐观？
   - 与公司历史估值、同业估值横向对比
   - 机构一致预期及目标价
   - "right price 很重要也不常见"——等待机会

### 第六步：可持续性判断 —— 四种好运

评估企业的长期可持续性，使用"四种好运"框架判断核心竞争力是否系统化、可复制：

| 好运类型 | 含义 | 企业实践示例 |
|----------|------|-------------|
| 第一种：完全随机 | 不可控、不可预测的运气 | 外部偶然事件催化（如名人自发传播、政策红利） |
| 第二种：数量保质量 | 多产生作品/产品，尝试越多成功概率越大 | 小批量测试 → 看市场反应 → 追加资源/砍掉 |
| 第三种：提升自身能力 | 持续提高专业水平，用能力保质量 | 内部研发体系 + 工业化优化 + 人才绑定机制 |
| 第四种：专家吸引力 | 成为领域专家/领导者后，机遇和资源主动找上门 | 品牌吸引力形成正反馈循环，头部资源主动汇聚 |

**判断标准**：
- 如果一家企业只依赖第一种好运（偶然事件），其成功不可复制
- 如果同时占据第二、三、四种好运，其核心竞争力是系统化的、可持续的
- "单个爆款有偶然性，但产品矩阵的持续产出能力已经形成"——这是可持续的标志

### 第七步：综合结论

输出结构化结论：

0. **市场现状快照**（优先来自 `tdx`，缺失项用 `tushareMcp` 补充）：**数据时效声明**（分析基准日、行情截止日、财报报告期、拉取时间）；当前价/涨跌、股价历史位置、当前估值（PE/PB/市值/股息率）、最新报告期核心财务指标（营收/净利/ROE/毛利率）、分红、公告/机构一致预期、大盘/宏观背景。每项标注 `trade_date`/`period`/`ann_date` 与来源；若某类数据滞后须显式说明。

1. **三要素判断**：
   - Right Business：是/否/待观察，核心理由
   - Right People：是/否/待观察，核心理由
   - Right Price：是/否/待观察，核心理由

2. **五维度评分**（0-10 分）：
   - 商业模式 ___/10
   - 市场规模 ___/10
   - 竞争壁垒 ___/10
   - 企业文化 ___/10
   - 估值 ___/10

3. **核心风险**：列出 2-3 个主要不确定性

4. **最终判断**：
   - 是否值得投资？
   - 安全边际如何？
   - 建议观察周期（"大家可以先看个五年十年吧"）
   - 持续性争议会存在多久？

## 公司知识库

`references/companies/` 是**用户私人沉淀的公司理解知识库**，与分析流程相互独立。

### 它是什么

- 用户自己长期积累的、对各家公司的深度理解
- 每家公司一个独立 `.md` 文件，整合创始人一手表达、投资人分析实例、自己的五维度评分
- 作为分析参照存在——已收录的公司，分析时可加载其理解作为背景

### 它不是什么

- **不是每次分析的产物**：日常分析直接用通用框架完成，输出报告即可，不必写入知识库
- **不是自动沉淀的**：只有当用户明确说"把这家公司沉淀/收录进知识库"时，才创建新文件
- **不是必须的**：分析未收录的公司时，通用框架完全够用

### 分析时如何使用

1. 查看 `references/companies/README.md` 索引，确认目标公司是否已收录
2. **已收录**：读取对应文件，将其中"创始人理解"和"投资人分析实例"作为分析参照（背景知识，不照搬结论）
3. **未收录**：直接用通用框架独立分析，无需做任何沉淀动作
4. **多公司对比**：可同时加载多个公司上下文做横向参照，但**不将一家公司的具体结论套用到另一家**

## 核心原则

- **数据以最新为准**：每次分析必须经 MCP 实时拉数并校验时效；禁止复用旧对话中的股价/财报数字，禁止未验证就写死 `period` 或 `trade_date`
- **充分阅读**：价值分析需要大量阅读公司资料（财报、访谈、研报）。"有几个发表各种观点的人是看过所有能找到的资料的？"
- **第一性原理**：从本质出发，持续用理性和事实逼近本质，而非一锤定音
- **长期视角**：投资买的是"未来的总量"（速度 × 时间），而非短期的"加速度"
- **承认不确定性**：持续性争议会长期存在，企业需要"一直证明下去"——但"他们自己不需要证明什么，只要继续做该做的事情就好"
- **做对的事情**：分析本身也要"做对的事情"——不为了热点而热点，不为分析而分析
- **独立思考**：独立判断每家公司，不将一家公司的结论机械套用到另一家
