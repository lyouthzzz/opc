# 市场数据手册：问财增强与默认检索降级

本手册记录 `价值投资` 与相关 skills 在「市场现状分析」环节的数据路径。同花顺问财是可选的结构化数据增强；没有问财 Token 时，改用 Agent 可用的网页/搜索工具、用户提供的数据与模型已有知识，不阻断定性分析。

---

## 一、问财可用时的技能分工

| Skill | 适合问题 | 默认用途 |
|-------|----------|----------|
| `hithink-market-query` | 个股 / ETF / 指数行情、涨跌幅、成交量、成交额、换手率、资金流、技术指标 | 个股与指数的默认入口 |
| `hithink-industry-query` | 行业估值、盈利、财务、行情、行业排名 | 行业口径与横向比较 |
| `hithink-sector-selector` | 板块涨跌幅、主力资金、估值分位、板块筛选 | 板块环境与主题热度 |
| `hithink-hkstock-selector` | 港股筛选、港股财务 / 行情条件组合 | 港股入口 |

原则：

- 个股、ETF、指数的最新行情，先用 `hithink-market-query`。
- 行业视角的问题，先用 `hithink-industry-query`。
- 板块和概念热度、资金流，先用 `hithink-sector-selector`。
- 港股问题优先带上代码和市场后缀，并用 `hithink-hkstock-selector`。
- 某项字段问财未稳定返回时，可切换到其他可靠来源，但必须明确标注来源、日期和口径；仍无法核验时说明缺失。

## 二、可选环境

只有选择使用问财增强时才需要以下环境变量：

```bash
export IWENCAI_BASE_URL="https://openapi.iwencai.com"
export IWENCAI_API_KEY="你的_IWENCAI_API_KEY"
```

未配置时不要要求用户补 Token，也不要停止分析；直接进入默认检索降级。已配置但 Agent 会话尚未加载新技能时，可重启 Agent 后使用问财。

## 三、数据时效性校验

### 3.1 三个时间锚点

每次分析都先确定：

| 锚点 | 含义 | 获取方式 |
|------|------|----------|
| 分析基准日 | 本次分析执行的日历日 | 当前日期 |
| 行情截止口径 | 行情 / 估值口径的最新时间 | 实际来源中的日期、更新时间或“最新/今日”口径 |
| 最新报告期 | 财务数据对应的报告期 | 实际来源中的报告期、季报/年报口径 |

### 3.2 校验规则

1. 行情类结果要能说明是“最新”“今日”或带明确日期；如果同一标的多个问句结果冲突，优先采用时间更新且字段更完整的一条。
2. 财报类结果必须能识别报告期；若只有数值没有报告期，不写成确定事实。
3. 行业 / 板块类结果优先要求“今日”“当前”“最新”。
4. 分红、回购、股东、机构预期等事件类结果，优先限定“最近一年”“最近90日”或“最新公告”。

### 3.3 标准声明

在市场现状快照开头固定输出：

```text
数据时效：分析基准日 YYYY-MM-DD | 行情截止 <来源口径/未实时核验> | 财报报告期 <最新报告期/未知> | 数据来源 <问财/通达信/网页检索/用户提供/模型知识> | 检索时间 YYYY-MM-DD HH:MM
```

## 四、标准查询模式

以下问句是模板，可用于问财或通用搜索，不要求逐字照抄，但必须保留时间口径与核心字段。

### 4.1 A 股个股市场现状

1. 大盘环境

```text
上证指数最新价、沪深300市盈率、今日A股成交额、今日涨跌家数
```

2. 个股行情与估值

```text
贵州茅台 600519.SH 最新价、涨跌幅、成交额、换手率、总市值、市盈率TTM、市净率、股息率
```

3. 历史位置与技术状态

```text
贵州茅台近1年涨跌幅、52周高点和低点、20日均线、60日均线、120日均线、MACD、RSI
```

4. 最新财报口径

```text
贵州茅台最新报告期营收、净利润、ROE、毛利率、净利率、同比增速
```

5. 资金与分红

```text
贵州茅台主力资金流向、近一年分红、股东户数变化、机构一致预期
```

6. 行业 / 板块环境

```text
白酒行业估值排名、白酒行业盈利情况、白酒板块今日涨跌幅和主力资金净流入
```

### 4.2 港股市场现状

```text
泡泡玛特 09992.HK 最新价、涨跌幅、总市值、市盈率、近一年涨跌幅、南向持股、最新报告期营收和净利润
```

如果单个问句返回不足，可拆成两到三个更短的问句。

### 4.3 行业 / 板块筛选

```text
A股行业估值排名
主力资金净流入前十的板块
港股科技股中市盈率低于30倍且营收增长为正的股票
```

---

## 五、问句改写规则

- 保留用户核心意图，不改变筛选条件。
- 口语化表达改成金融术语，例如“最近怎么样”改成“最新价、涨跌幅、成交额、资金流向”。
- 复合问题拆成少量独立问句，避免一个问句里塞入过多条件。
- 若首次返回为空，最多放宽两次条件，并在结果中说明最终使用的问句。

---

## 六、输出要求

- 所有市场数据都注明实际来源；使用模型已有知识时标注“未实时核验”。
- 能识别时间口径时，必须写出时间口径。
- 某项字段拿不到时，写“当前可用来源未能核验该字段”。
- 不得把历史知识库里的旧数字当作实时数据。

---

## 七、边界与错误处理

| 情况 | 处理方式 |
|------|----------|
| 返回为空 | 放宽条件重试，最多 2 次 |
| 返回字段不全 | 拆分问句，优先保留核心字段 |
| 多个问句结果冲突 | 采用时间更新、字段更完整的一条，并标注冲突 |
| 港股 / 财报字段不稳定 | 明确标注缺失，不补猜 |
| 技术分析必需字段缺失 | 输出“数据不足” |

### 默认检索降级

问财或通达信不可用时：

1. 优先使用 Agent 的网页/搜索工具查找交易所、公司公告、定期报告、指数公司及可靠财经来源。
2. 其次使用用户提供的行情或财务数据，并明确标为“用户提供”。
3. 模型已有知识只用于商业模式、产业链和历史背景等相对稳定内容，不得作为当前股价、最新财报或公告事实。
4. 无法核验实时字段时继续输出定性框架，但不提供伪精确买卖价、估值或技术指标。

> 问财数据只用于辅助形成市场现状与估值判断，不替代对商业模式、管理层和长期竞争力的定性理解。
