# 股票比较

把几只股票放在同一张资金使用表里比较：哪只更值得占用资金，哪只该等，哪只该卖给更好的机会让位。

## 必须先加载

执行前读取并遵循：

- `skills/价值投资/SKILL.md`
- `skills/股票魔法师/SKILL.md`

不要复制两套框架；用它们分别产出长期质量判断和交易计划。

## 输入

用户至少给出几只股票代码或名称。若要给出换仓计划，优先收集：

- 当前持仓、买入成本、仓位金额或股数
- 可用现金
- 交易周期：短线、波段、中长期
- 单笔最大可承受亏损

缺失也继续做第一版，但把仓位建议降级为比例区间或“数据不足”。

## 流程

1. **锚定数据时效**：按 `价值投资` skill 的要求确定分析基准日、最近交易日、最新报告期；实时拉取或核验行情、财报、估值、成交量、均线、52 周高低点、相对强度和大盘/板块环境。
2. **逐只股票分析**：
   - 用 `价值投资` 判断 Right Business / Right People / Right Price 和核心风险。
   - 用 `股票魔法师` 判断趋势模板、VCP/中枢点、成交量、买点、止损、持有或卖出信号。
3. **先剔除**：
   - 看不懂生意且没有明确技术优势的，剔除。
   - 基本面恶化且技术面破位的，优先卖出或不买。
   - 技术面未触发且估值不便宜的，观察。
4. **机会成本排序**：只比较通过基础筛选的股票，按以下顺序排：
   - 好公司 + 合理/低估 + 技术触发
   - 好公司 + 合理/低估 + 技术未触发
   - 一般公司 + 强技术触发
   - 好公司但明显高估或破位
   - 看不懂、重大风险或数据不足
5. **资金计划**：
   - 先定卖出/止损，再谈买入。
   - 买入仓位按 `股票魔法师` 的单笔风险公式约束。
   - 换仓只在新机会的预期风险收益明显优于原持仓时给出；不为“看起来更热”换仓。

## 输出

先给结论，再给表格和执行价位：

```text
结论：买入 / 持有 / 减仓 / 卖出 / 换仓 / 观察 / 数据不足

机会成本排序：
1. 股票A：最值得占用资金
2. 股票B：等待触发
3. 股票C：资金占用效率低

执行计划：
- 买入：代码、触发价/中枢点、买入区间、止损位、计划仓位、加仓条件
- 持有：移动止损、减仓条件
- 卖出/换仓：卖出条件、换入对象、换仓比例

逐只依据：
- 价值：Right Business / Right People / Right Price
- 技术：趋势、形态、成交量、相对强度
- 风险：最重要的 1-3 条
- 失效条件：什么发生后计划作废
```

数据不足时，不猜关键价位；列出完成判断所需的最少缺失字段。

不构成投资建议，不承诺收益。
