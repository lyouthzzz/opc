# 行业分析

## 可选数据增强与降级策略

开始分析时只检查可用性，**不得打印、回显、记录或在回答中暴露 Token**。通达信和同花顺问财都是可选增强，不是使用本 skill 的前置条件：

- **通达信增强**：若 `TDX_API_KEY` 已配置且官方通达信 MCP 可用，优先用于结构化行情查询；未配置、已跳过或连接失败时，继续分析。
- **问财增强**：若 `IWENCAI_API_KEY`、`IWENCAI_BASE_URL` 及所需问财 skills 可用，优先用于行情、财务、行业和公告查询；未配置、已跳过或不可用时，继续分析。

降级时按以下顺序取数：用户提供的数据 → Agent 可用的网页/搜索工具 → 模型已有知识。涉及当前股价、估值、财报、公告或技术指标时，优先检索并标注来源与日期；若无法实时核验，明确写“未实时核验”或“数据不足”，不得把模型记忆当成当前事实，也不得编造精确价位。不要因缺少 Token 反复要求用户配置；只有用户主动需要更稳定的结构化数据时，才简短提示可重新运行安装向导。

用这个 skill，把“行业感觉”压成一套可复查的行业机会判断。

先用当前可用的可靠来源获取行业、板块、个股数据，再做行业轮动、产业链和个股筛选。问财可用时优先使用；不可用时走默认检索降级。不要跳过数据时效，也不要把旧逻辑当现状。

## 必须先读

执行前先读取并遵循：

- `../价值投资/references/market-data-guide.md`

这份手册已经定义了问财增强、默认检索降级、时间锚点、问句模式和缺失字段处理。这里不重复发明一套。

## 默认工具顺序

优先按下面顺序取数：

1. `hithink-industry-query`
   - 行业景气、估值、盈利、行业排名、行业财务
2. `hithink-sector-selector`
   - 板块涨跌幅、主力资金、主题热度、板块筛选
3. `hithink-market-query`
   - 上证/沪深300等指数、大盘成交额、个股行情、资金流、技术状态

可选增强：

- `行业轮动监控`
- `产业链解读`
- `捕捉公司事件机会`

这些社区 skills 如果已安装就用；没装就继续用当前可用来源 + 手工归纳，不要卡住。

## 触发场景

当用户要做以下事时使用：

- 找未来 6 到 12 个月有机会的 A 股行业或板块
- 判断当前宏观周期 / 利率环境下最可能跑赢的行业
- 解读某行业产业链、利润弹性和卡脖子环节
- 梳理近 30 天某行业的政策、订单、涨价、并购、扩产催化
- 从某行业里筛选基本面改善但股价还没大涨的 A 股公司

## 总流程

### 第零步：先锚定时效

严格按 `market-data-guide.md` 的规则先确定：

- 分析基准日
- 行情截止口径
- 最新报告期

所有问句优先带上 `最新`、`今日`、`当前`、`近30天`、`最近一年` 这类时间口径。

输出里固定带一句：

```text
数据时效：分析基准日 YYYY-MM-DD | 行情截止 <来源口径/未实时核验> | 财报报告期 <最新报告期/未知> | 数据来源 <问财/通达信/网页检索/用户提供/模型知识> | 检索时间 YYYY-MM-DD HH:MM
```

### 第一步：先看宏观，再谈行业

如果问题带有“未来 6 到 12 个月”“按宏观周期和利率环境”这类表述，先判断宏观状态：

- 增长：PMI、工业增加值、消费、投资、出口
- 通胀：CPI、PPI
- 流动性：社融、M2、利率
- 风险偏好：大盘成交额、宽基指数强弱、北向或主力资金

可用问句模板：

```text
最新 PMI、CPI、PPI、社融、M2、10年期国债收益率、A股成交额、沪深300最新表现
```

先给出一个简短宏观判断：

- 复苏早期 / 复苏中期 / 滞胀 / 去库存 / 宽松 / 紧缩

然后再映射到可能受益行业。不要一上来就报行业名单。

### 第二步：行业轮动初筛

先在行业和板块层面做横向比较，不急着落个股。

至少查这四类：

1. 行业景气或盈利改善
2. 行业资金持续流入
3. 行业估值分位或估值排名
4. 行业近 1 个月、近 3 个月相对强弱

可用问句模板：

```text
A股行业最新估值排名、行业盈利排名、行业涨跌幅排名
主力资金持续净流入的板块
近3个月跑赢沪深300的行业
景气度改善且估值不高的行业
```

然后给每个候选行业做一个最简单的五项打分，每项 0-2 分，总分 10 分：

- 景气改善
- 资金流入
- 估值位置
- 政策/产业催化
- 机构预期或盈利上修

`ponytail:` 先用这个 10 分表够了，只有用户明确要量化模型时才扩展。

### 第三步：确定重点行业

对于得分靠前的 3 到 5 个行业，继续回答：

- 为什么现在轮到它
- 这个逻辑更偏宏观、政策、价格、订单还是技术迭代
- 最关键的验证指标是什么
- 哪个条件失效会推翻判断

如果用户问“当前最可能跑赢的 3 个行业”，默认只给 3 个，不扩成 10 个。

## 专项任务

### A. 未来 6 到 12 个月哪些行业景气改善、资金持续流入、估值仍处中低位

流程：

1. 先给宏观状态
2. 再列行业候选池
3. 用 10 分表排序
4. 输出前 3 到 5 个行业

输出至少包含：

- 行业名
- 核心驱动
- 当前估值位置
- 资金面结论
- 未来 6 到 12 个月的主要验证点
- 最大反证

### B. 按照宏观周期和利率环境，当前最可能跑赢的 3 个行业

先写：

- 当前宏观阶段
- 利率方向或流动性状态
- 为什么这会利好某类行业

再给 3 个行业，每个行业只写最关键的 3 条理由，别铺满。

### C. 请解读[某行业]产业链，找出利润弹性最大和卡脖子环节

优先用 `产业链解读`；没有就自己拆：

1. 上游原材料 / 设备 / 核心零部件
2. 中游制造 / 集成
3. 下游应用 / 渠道 / 客户

必须回答这四件事：

- 产业链结构
- 利润主要沉淀在哪一层
- 利润弹性最大的环节
- 卡脖子 / 供给瓶颈 / 国产替代关键环节

如果能拿到经营数据，再补：

- 哪类公司最容易受益于涨价、放量、降本或渗透率提升

### D. 近30天[某行业]有哪些政策、订单、涨价、并购、扩产催化

优先用：

- `新闻搜索`
- `公告搜索`
- `研报搜索`
- `事件数据查询`
- 可选 `捕捉公司事件机会`

时间窗写死为近 30 天，不要偷换成“近期”。

催化按这五类归档：

- 政策
- 订单
- 涨价
- 并购/重组
- 扩产/资本开支

每条催化都尽量写：

- 日期
- 事件
- 影响方向
- 受益环节

不要把没有日期的传闻写成已验证催化。

### E. 在[某行业]中，筛选营收增速改善、ROE较高、机构预期上修、股价未大涨的A股公司

先明确行业范围，再做四层筛选：

1. 行业内 A 股公司池
2. 最新报告期营收增速改善 / 净利润改善
3. ROE 较高
4. 机构评级、盈利预测或目标价上修
5. 股价未大涨：可用近 1 个月 / 3 个月涨幅不过热，或距离 52 周高点仍有空间

可用问句模板：

```text
[某行业] A股公司，最新报告期营收增速改善、ROE较高、机构预期上修、近3个月涨幅不大
```

如果一个问句字段不全，就拆开查，不要为了省步骤牺牲结果可用性。

输出至少包含：

- 代码 / 名称
- 所处细分环节
- 最新报告期营收或利润改善点
- ROE
- 机构预期变化
- 近期股价表现
- 入选原因

```text

```

## 约束

- 不把宏观观点直接当结论，必须落到行业数据
- 不把行业热度直接当机会，必须看估值和盈利
- 不把旧研报或旧叙事当现状，必须校验时间
- 数据不足时直接写 `数据不足`，并列最少缺失字段
- 不构成投资建议
